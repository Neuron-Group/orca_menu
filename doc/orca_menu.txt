*orca_menu.txt*    Bottom menu plugin for Neovim

INTRODUCTION ~

Orca Menu provides a bottom menu bar for Neovim using floating popup windows.
It supports keyboard navigation, nested submenus, mouse interaction, lualine
integration, custom key hints, and Lua callbacks.
Tall popup menus automatically keep the selected row visible, and child
submenus open next to the visible parent row.

REQUIREMENTS ~

    Neovim 0.9+
    lualine.nvim
    hydra.nvim        For the default Hydra-backed menu mode.

COMMANDS ~

    :OrcaMenu           Toggle menu mode
    :OrcaMenu {index}   Open top-level menu {index}

SETUP ~

>lua
require('orca_menu').setup({
  enable_mouse = true,
  keys = {
    open = '<M-m>',
    mode_backend = 'hydra',
  },
  lualine = {
    section = 'y',
    spacing = ' ',
  },
  topbar = {
    hint_format = '{label}({hint})',
  },
  submenu = {
    border = 'rounded',
    min_width = 18,
    scroll_indicator_up = '↑',
    scroll_indicator_down = '↓',
  },
  highlights = {
    menu = 'NormalFloat',
    menu_sel = 'OrcaMenuSelected',
    accelerator = 'OrcaMenuHint',
  },
  menus = {
    {
      label = '&File',
      key = 'f',
      items = {
        { label = '&Write', key = 'w', command = 'write' },
        { label = 'Write &Quit', key = 'q', command = 'wq' },
      },
    },
  },
})
<

LSP OVERRIDES ~

Orca Menu can apply config overrides when exact LSP client names are active.

>lua
require('orca_menu').setup({
  menus = {
    { label = '&File', key = 'f', items = {} },
  },

  lsp_overrides = {
    rust_analyzer = {
      menus = {
        {
          label = '&Rust',
          key = 'r',
          items = {
            { label = '&Run', command = 'RustLsp runnables' },
          },
        },
      },
    },

    lua_ls = {
      topbar = {
        hint_format = '{hint} -> {label}',
      },
    },
  },
})
<

Behavior:

    exact client.name matching
    matching overrides come from LSP clients attached to the current buffer
    menus replace base menus completely
    other fields are deep-merged
    current buffer or LSP changes close popups and exit menu mode before refresh

DEFAULT KEYS ~

    <M-m>       Enter menu mode
    h / <Left>  Previous top menu
    l / <Right> Next top menu
    j / <Down>  Next popup row
    k / <Up>    Previous popup row
    <CR>        Select current item
    <BS>        Go back
    <Esc>       Go back or close top-level popup
    q           Close menu mode

ESC BEHAVIOR ~

    child submenu       Close only that child and return to parent.
    first-level popup   Close the popup and leave menu mode.
    after action        Close all popups and leave menu mode first.
    window resize       Close all popups and leave menu mode.

TOP MENU FIELDS ~

    label       Visible text. Prefix a letter with '&' for accelerator.
    key         Optional direct key for the visible top menu.
    items       Top-level popup entries.

TOP BAR HINT FORMAT ~

Top-bar key hints can be customized with topbar.hint_format.

Supported placeholders:

    {label}    Top-level menu label
    {hint}     Rendered key hint

Examples:

>lua
require('orca_menu').setup({
  topbar = {
    hint_format = '{label}[{hint}]',
  },
})
<

Produces:

    File[f]

Other useful formats:

    {label}({hint})    -> File(f)
    {hint} -> {label}  -> f -> File

For fully custom behavior, topbar.hint_format may also be a function:

>lua
require('orca_menu').setup({
  topbar = {
    hint_format = function(ctx)
      return string.format('%s <%s>', ctx.label, ctx.hint)
    end,
  },
})
<

POPUP ITEM FIELDS ~

    label       Visible text. Prefix a letter with '&' for accelerator.
    key         Optional direct key for the visible popup item.
    command     Ex command to run.
    keys        Normal-mode keys to feed.
    action      Lua function callback.
    lua         Lua function or Lua string.
    items       Nested submenu items.

Use { label = '-' } to insert a separator.

KEY HINTS ~

Popup items can display a right-aligned key hint column.

Examples:

    1
    Tab
    Space
    Ctrl+x

HIGHLIGHTS ~

    menu            Base popup highlight
    menu_sel        Selected popup row highlight
    accelerator     Popup key-hint highlight

Default internal groups:

    OrcaMenuSelected
    OrcaMenuHint

MOUSE ~

Left click works on the top bar and on visible popup entries.
Mouse wheel scrolls the hovered open popup menu by nearly a full page.
The popup border shows ↑ or ↓ on the right side when more items are hidden.
Those markers can be changed with submenu.scroll_indicator_up and
submenu.scroll_indicator_down.
Set enable_mouse = false to disable Orca Menu mouse bindings entirely.
With the Hydra backend, opening a menu via mouse also enables keyboard
navigation, and closing it via mouse exits menu mode.

    top bar, same item      Close the open popup tree.
    top bar, other item     Switch to that popup.
    outside all popups      Close the whole popup tree.
    popup item, no child    Run the item and close all popups.
    popup item, has child   Open its child popup.

LUALINE ~

The menu bar is rendered by registering components into the configured lualine
section.
